## Security Features

âœ… **Token Security**
- 64-character hex tokens (256 bits of randomness)
- Stored hashed in database
- One-time use only
- 1-hour expiration

âœ… **Rate Limiting**
- 2-minute cooldown between requests
- Prevents spam and brute force

âœ… **Email Enumeration Protection**
- Same response for valid/invalid emails
- Prevents discovering valid email addresses

âœ… **Auto-Invalidation**
- Old unused tokens invalidated on new request
- Only one active token per customer

âœ… **Password Security**
- Minimum 8 characters
- bcrypt hashing with salt
- Validated on both client and server
